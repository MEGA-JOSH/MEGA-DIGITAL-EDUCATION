Version Control with Git

Duration: 2 hours

Objective

Learn to manage code changes and collaborate with others using Git.

Topics Covered

  • What is Version Control?

    Version control is a system that records changes to files over time, allowing you to revert to previous versions.

  • Introduction to Git

    Git is a distributed version control system that helps track changes in source code.

  • Basic Git Commands

    Common commands include:

    • git init: Initializes a new Git repository.
    • git clone: Clones an existing repository.
    • git add: Stages changes for commit.
    • git commit: Saves changes to the local repository.
    • git push: Sends committed changes to a remote repository.
    • git pull: Fetches and merges changes from a remote repository.
  • Branching and Merging

    Branches allow for separate lines of development. Learn how to create and merge branches.

    
git checkout -b new-feature  // Create and switch to a new branch
git merge new-feature         // Merge changes from new-feature branch
  • Resolving Merge Conflicts

    Merge conflicts occur when changes from different branches conflict. Learn how to resolve these conflicts.

Project

Set up a Git repository for a web project and practice using Git commands.

Summary

This section covered the basics of Git and version control, including essential commands and branching strategies.

Exercises

  • Initialize a new Git repository for a personal project.
  • Practice making commits and branching.
